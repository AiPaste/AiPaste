# Luminous Stack

AiPaste 的图标不应该走“真实剪贴板拟物”或“通用文档图标”那条平庸路径，而应该把这个产品最强的感知翻译成一个紧凑、可瞬时识别的符号：历史被整理成层，粘贴动作像一道被精密控制的能量流，从一个稳定的容器中被抽取出来。形式上，这意味着主体必须是单一的、记忆点极强的中央信号，而不是一堆零散的小隐喻。整个图标要看起来像经过无数次推敲后的结果，边界、节奏、比例都必须带着克制而昂贵的完成度。

空间结构上，图标要采用“外部柔和、内部锐利”的组织方式。底层是厚实、圆润、带体积感的应用载体，中层是稳定的文档或卡片系统，顶层则是一道更轻、更明亮的粘贴手势。这样的层次能让用户一眼感知到“剪贴板 + 历史堆栈 + 取出内容”三个动作被压缩进一个形状里。构图必须高度居中，留白被精确控制，让图标在 Dock、Finder、菜单切换器里都能保持一眼识别。

色彩不走系统默认蓝白模板，而是采用夜色底座配合电蓝到青绿的光场，再用一处暖橙作为操作能量点。冷暖之间的反差负责唤起“高速、可靠、带科技感”的印象，同时仍然保持专业工具的克制。高光要像玻璃和阳极氧化金属之间的混合材料，不能廉价，不能糖果化，必须呈现那种经过长时间手工打磨后才会得到的精密表面。

形体语言上，卡片不做真实纸张边缘，而是做略带浮起感的整齐叠层；粘贴动作不画箭头，不写字，而用一个抽象但清晰的“拉出”手势来表达。这个手势需要足够简练，像工业符号，又带一点运动轨迹的流动感。所有圆角、阴影、轮廓线都要像是顶级图标设计师反复校准后的结果，宁可少一笔，也不能多一笔，整体必须显得非常熟练、非常自信。

最终成品应该在 1024 尺寸下足够精致，在 32 或 16 尺寸下依然保留清晰轮廓。它不依赖文字，不依赖复杂细节，而依赖精确的形体关系和极高完成度来建立品牌识别。看上去要像一件被耐心雕刻出来的小型工业艺术品，而不是一个“表示功能”的通用按钮。
